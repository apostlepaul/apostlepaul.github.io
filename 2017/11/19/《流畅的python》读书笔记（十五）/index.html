《流畅的python》读书笔记（十五）

发表于 2017-11-19 | 分类于 reading notes | 本文总阅读量 次

第十七章讲期物第十八章讲并发，这两章读的比较艰辛，因为没有相关的基础，并且平时的工作也不涉及。所以，就先把能理解的东西记下来吧。

第17章 使用期物处理并发

期物指一种对象，表示异步执行的操作。

1.使用concurrent.futures实现并发

最简单的用法，实例化futures.ThreadPoolExecutor类，然后用TreadPoolExecutor.map让每个executor取执行相关的函数。

如果要在并发过程中实现更复杂的逻辑，可以将ThreadPoolExecutor.map展开成exectuor.submit()、futures.as_completed()、futrue.result()三个过程：

其实ThreadPoolExecutor并不是并行计算（各cpu都在跑）因为有GIL（全局解释器锁，一次只允许使用一个线程）的存在之所以变快是因为IO密集，标准库中所有执行阻塞型IO操作的函数，在等着操作系统返回结果时都会释放GIL，另外一个线程可以运行了，所以省去了等IO的时间。如果要实现CPU密集型的并行计算，只需要把ThreadPoolExecutor改成ProcessPoolExecutor

小插曲，使用TQDM包可以显示进度条。

futures.as_completed惯用法：构建一个字典，把各个期物映射到其他数据上，例如国家名，这样即使返回的结果顺序是乱的，也便于后续处理。

2. 不适用concurrent.futures的情况

如果ThreadPoolExecutor处理某个作业不够灵活，可能要用treading模块中的某个组件自行定制方案。
同样，ProcessPoolExecutor不够灵活的时候，要用mutiprocessing模块。

第18章 使用asyncio包处理并发

显示文本式动画的诀窍：使用退格符（\x088)把光标移回来。

1.线程与协程对比

线程要用threading.Thread声明线程要跑的job(target=xx, args=(xx,xx)),用start()开始，用join()等待结束。

其中done是一个信号，控制Thread的结束。
协程要用asyncio.async()排定spin协程的运行时间，不需要start，在要结束的时候，用cancel()结束，目标job catch asyncio.CancelledError来break。主函数里，要用loop=asyncio.get_event_loop()得到事件loop, 然后run_until_complete()来跑协程排好的job。

以上可以演示异步程序的运行，即spinner 和slow_function是并行的。这样的效果，可以用threading多线程来实现，也可以用asyncio异步协程来实验，并且后者是更优雅的实现。
yield from asyncio.sleep(DELAY)不会阻塞主线程
asyncio.async统一了协程和期物。
通过res = yield from foo()从协程获得结果。

2. 使用asyncio和aiohttp实现异步请求

实现http的异步请求，aiohttp是常用的包。
get_flag不能直接用request包的那个request()了，因为这个不是异步的，会阻塞线程，要用aiohttp.request().主循环直接看代码吧：

wait会把协程列表包成Task，变成了Future类实例，然后loop.run_until_complete(…)会去驱动这些实例。

非常需要注意的是：

使用asyncio包时，我们编写的异步代码中包含由asyncio本身驱动的协程（即委派生成器），而生成器最终把职责委托给asyncio包或第三方库（如aiohttp)中的协程。

没有明白asyncio.wait()跟asyncio.async()的区别，好像是，后者是用来跑类似loop循环这样的task，像那个spinner。

3. 比run_until_complete()更精细的控制

loop.run_until_complete()会等所有协程跑完，返回结果。
如果要做更精细的控制，比如每跑完一个协程，就更新一下进度条，那么要用asyncio.as_completed()函数。（跟futures.as_completed()对应的）
而ThreadPoolExecutor里的max_workers在asyncio里也有对应的版本，即asyncio.Semaphore，它内部维护了一个计数器，表示可用的协程数，根据正在跑的协程动态改变，计数器为0时，会阻塞新的协程。
