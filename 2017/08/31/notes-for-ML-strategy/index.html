Tips for ML strategy

发表于 2017-08-31 | 分类于 tech notes | 本文总阅读量 次

听deeplearning.ai的ml strategy course记录下来的一些点，常看常新。

orthogonalization

  1. fit training set well on cost function
    1. fit dev set well on cost function
    2. fit test set well on cost function
    3. performs well in real word

主要的思想是，在不同的阶段，调整不同的knob。
一个Knob是一个正交的调整项

single number evaluation

在建模初期，设定一个单一的指标，能够加速迭代。

satisfying and optimizing

区分要优化的指标和限定指标

Train/dev/test distributions

dev set要和test set同分布。
在工作的一开始就明确目标很重要。

comparing to human-level performance

training error和bayes error直接的差别叫avoidable bias。training error和dev error的差别叫variance。先优化avoidable bias, 再优化variance。
优化avoidable bias的方法：

  • train bigger model
  • train longer/better optimization algorithms(momentum, RMSprop Adam)
  • NN architecture/hyperparamerters search（RNN，CNN)

优化variance的方法：

  • more data
  • regularization
  • NN architecture/hyperparamerters search

error analyse

要观察分析被模型mislabel的case
要考虑数据label有误的情况

how to start a new ml project

  1. set up dev/test set and metrix
  2. build initial system quickly
  3. user Bias/Variance analysis and Error analysis to prioritize next steps

mismatched training problem

经常发生训练数据集跟测试数据集来自于不同的分布。
把不同分布的数据全部混在一起，再去划分training set和dev set，这样的做法不好。更好的做法是，dev set就是从目标分布中取的。这时会发生mismatched training problem。
要先量化分析mismatched error：training-dev error跟dev error直接的gap是比较公平的评估指标。
解决mismatched error的途径：

  1. 人工分析分布的区别, error analysis
  2. 想办法改变training set: artificial data synthesis

Transfer learning

简单来说，就是拿隐层出来。前提是，pre-train的目标是学习一些low level的特征。

multi-task learning

什么时候multi-task是合理的？

  1. task share lower-level features
  2. amount of label for each task is similar
  3. train a big enough neural network to do well on all the tasks

end-to-end deep learning

by pass中间步骤，但是要结合数据量考虑，也许有些任务分解成两个小步骤，各个小步骤都会有更多的数据和效果
