发表于 2017-10-17 | 分类于 reading notes | 本文总阅读量 次

讲的是面向对象编程比较抽象和底层的东西，不是完全理解，只整理自己觉得有用的点。

第11章 接口：从协议到抽象基类

不要自己去实现抽象基类。
可以认为python里的接口有两类：

  • 动态协议：鸭子类型的依赖，由于其动态特性，猴子补丁也是可以生效的。
  • 抽象基类：ABC(Abstract Base Class), 能使接口更明确，能验证实现是否符合规定。

我理解所谓的“猴子补丁”：先声明了对象，后给类添加补丁（新的方法），已经声明的对象也是可以用这些补丁的。

1. 序列协议

python中的序列协议实现的很好。

  • 自定义的类中只要实现了__getitem__方法，即使没有实现__iter__和__contains__方法，python也会通过调用__getitem__让迭代和in运算符可用。
  • 可变的序列协议需要实现__setitem__方法

2. 抽象基类

概念

  • 定义抽象基类的子类，必须要实现基类中的抽象方法。抽象方法在__abstractmethods__里可以看。
  • 继承了抽象基类，就可以直接用基类中定义的方法。
  • 标准库中的抽象基类在collections.abc中。其中Callable和Hashable比较特殊，主要是为isinstace方法提供支持，以一种安全的方式判断对象能不能被调用或散列。官方文档

实现

  • 自己定义的抽象基类要继承自abc.ABC。需要注意不同的python版本继承abc.ABC的句法不一样。
  • 抽象方法用abc.abstractmethod装饰。@abc.abstractmethod和def之间不能有其他装饰器。

虚拟子类

  • 虚拟子类是不直接继承自抽象基类的子类。python不会检查虚拟子类是否符合抽象基类里的实现。虚拟子类也不会从抽象基类继承方法或属性。它就是注册了一下，会被判断为基类的子类。但是，我们需要保证实现，不然运行的时候会报错。
  • 需要用register注册，可以是函数也可以是装饰器。句法：在class上一句写@Tomobla.register，或Tombola.register(xxx)
  • 实现了__iter__、__contains__、和__len__就被注册到了抽象基类Iterable、Container、Sized上了。因为这三个抽象基类实现了__subclasshook__这个方法。
