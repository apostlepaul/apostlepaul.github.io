平时用到上下文管理器的机会并不多，但是这里也有一些挺有趣的特性。

第15章 上下文管理器和else块

1. else块

else子句不仅能在if语句中使用，还能在for、while和try语句中使用。

  • for/else: 仅当for循环没有被break语句终止，运行完毕后，才运行else块。
  • while/else: 仅当循环条件为假值而退出，没有被break打断，才运行else块。
  • try/else: try中没有异常抛出时，运行else。

    如果异常或者return、break或coutinue语句导致控制权跳到了符合语句的主块之外，else也会被跳过。

2. 上下文管理器和with块

上下文管理器对象存在的目的是管理with语句。
with语句的目的是简化try/finally模式。
上下文管理器协议包括__enter__和__exit__两个方法。在一个类上实现了这两个方法，那么这个类就是上下文管理器类。
而且这两个方法也可以手动调用。
其实也没那么神秘，就是在with了管理器类之后，其\_\_enter\_\_方法会被调用，离开with块的时候，\_\_exit\_\_方法会被调用。跟手动调用的效果一样的。

3. contextlib模块

标准库中的contextlib模块，有

  • closing
  • suppress
  • @contextmanager
  • ContextDecorator
  • ExitStack

是干什么用的都不知道。感觉也用不到。
@contextmanager是比较常用的，可以直接把一个函数变成上下文管理器的对象。其特点是：

  1. 用一个yield语句把代码分成两部分；
  2. yield之前的相当于__enter__的内容；
  3. yield之后的相当于__exit__的内容；
  4. 要把yield语句放在try/finally语句中。
