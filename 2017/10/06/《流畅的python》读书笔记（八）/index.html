进入本书的第三部分，面向对象惯用法。

第8章 对象引用、可变性和垃圾回收

本章先将python的变量的本质，再讲对象标识，值和别名等概念，接着讲引用和函数参数，最后讲垃圾回收，弱引用。

1. 变量不是盒子

python中的变量，是引用式变量。
python中的赋值语句应该先从右边读，右边创建了对象，才在左边绑定了变量给对象，就像给对象贴上了一个标签，这个标签就是变量。

2. 标识、相等性和别名

赋值运算要区别可变对象和不可变对象。简单来说，创建两个值相等的非引用类型对象（基础类型或扁平类型），是同一个对象；创建两个值相等的引用类型对象，是不同的对象：

可以把标识，也就是id()返回的值，理解为对象在内存中的地址。
==用来比较对象的值；is用来比较对象的标识。不同的对象可能有相同的值。
元组的不可变，是其内保存的对象的标识不可变，如果这些标识本身是可变的，那么元组是挡不住其变化的。
相同的整数和字符串，创建的对象是同一个，这个叫“驻留interning”。需要注意的是Cython不会驻留所有的整数和字符串。

3. 复制

列表的复制默认是浅复制，既，复制了最外层的容器，副本中的元素是源容器中元素的引用。例如:

之所以对list += 之后l1[1]和l2[1]仍是相同的对象，是因为list是可变对象，+=之后其地址不会发生变化。由于tuple是不可变对象，所以对tuple进行+=之后，所以l1[2]是一个新的对象了，就跟l2[2]不再是同一个对象了。

但是元组的复制不是，元组的复制跟起了个别名是一样的。
字符串和整数的

可以用copy模块的copy()和deepcopy()来进行浅拷贝和深拷贝。为了防止深拷贝太深，还可以给对象实现__copy__()和__deepcopy__()来控制copy()和deepcopy()的行为。

4. 形参

python唯一支持的参数传递模式是共享传参（call by sharing）。共享传参指函数的各个形式参数获得实参中各个引用的副本。也就是说，函数内部的形参是实参的别名。

所以函数可能会修改作为参数传入的可变对象。

不要使用可变对象作为函数的默认形参，因为函数参数的默认值是在模块加载时计算的，这样所有以默认值作为参数的函数，共享的是同一个可变对象，会造成预期外的结果。

如果函数内部会改变某一形参，而这个形参又是可变对象时，要小心确定这样做是不是符合预期的。如果不想改变这个外部的可变对象，那么久在函数内部做一个拷贝。同时要注意浅拷贝和深拷贝的问题。

5.弱引用

del语句删除名称，而不是对象。删除对象的最后一个引用（名称），或对象不可获取时，会把对象当做垃圾回收。
有时候需要引用对象，但不增加对象的引用数量，这就是弱引用，在python中可以用weakref模块。
weakref.ref()是低层接口，最好用weakref集合(WeakKeyDictionary/WeakValueDictionary/WeakSet)和finalize。
弱引用的使用场景呢？例如如果一个类想要记录所有实例，就可以用弱引用来实现。
基本的list和dict实例是不能作为弱引用的所指对象的。而其子类可以。set可以。int和tuple,子类也不行。
