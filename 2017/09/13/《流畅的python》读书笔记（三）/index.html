第3章 字典和集合

泛映射类型

标准库里所有映射类型都是利用dict来实现的。
只有可散列的数据类型才能用作这些映射里的键。
可散列对象：

  • 生命周期内散列值不变
  • 实现__hash__()方法
  • 实现__qe__()方法

通常情况下id()返回的值就是对象的散列值。

映射的弹性键查询

defaultdict

如果在创建defaultdict的时候没有指定defatulfactory, 查询不存在的键会触发KeyError。
default_factory只会在\_getitem__里被调用。在其他方法，如dd.get(k)时会返回None。
其背后是__missing__的功劳

特殊方法__missing__

__missing__方法只会被__getitem__调用。

字典的变种

  • collections.OrderedDict: 保持添加键的顺序，popitem会删除并返回最后一个，popitem(last=False)会从头pop
  • collections.ChainMap: 可以容纳数个不同的映射对象
  • collections.Counter: 给键准备了一个计数器。most_common(n)会返回出现次数最多的n个键。
  • collections.UserDict: 把标准dict用纯python又实现了一遍。让用户继承写子类的。

子类化UserDict

UserDict有一个叫做data的属性，是dict的实例，是最终存储数据的地方。
python3.3在types模块引入了一个封装类，叫MappingProxyType, 它返回的是一个只读的映射视图。如d={}可以改变，d_proxy=MappingProxyType(d)，d_proxy是不能改变的。

集合论

也有集合推导。
操作很丰富，基本跟数学符号一致。

dict和set的背后

set和dict的核心就是Key的散列。
由此导致的特点：

  • 键必须是可散列的。
  • 字典在内存上的开销巨大。
  • 键查询很快。
  • 键的次序取决于添加顺序。
  • 往字典里添加新键可能会导致键的顺序改变，所以不要对字典同时进行迭代和修改。
